# Snake Game для ESP32

Класична гра Snake для ESP32 з OLED-дисплеєм SSD1306, двома кнопками, збереженням рекорду та оновленою архітектурою з тестованим ядром логіки.

## Огляд

Проєкт працює на ESP32 через PlatformIO та Arduino framework. Код поділено на три зрозумілі шари:

- `SnakeGame`: оболонка застосунку з обробкою кнопок, станами гри та збереженням даних.
- `snake_core`: чиста ігрова логіка, яку використовує і прошивка, і автотести.
- `snake_render`: рендеринг на OLED та анімації екранів.

Такий поділ спрощує підтримку проєкту і дозволяє тестувати головні правила гри без ESP32-заліза.

## Ігровий процес

### Особливості
- Ігрове поле: `32 x 14` клітинок на OLED `128 x 64`.
- Керування двома кнопками: поворот ліворуч і праворуч.
- Випадкова їжа зі спавном без накладання на змійку чи перешкоди.
- Випадкові перешкоди при кожному reset.
- Анімований splash, миготіння при зіткненні та екран Game Over.
- Збереження рекорду в NVS ESP32 через `Preferences`.
- Скидання рекорду довгим натисканням двох кнопок на splash-екрані.

### Керування
- Ліва кнопка: `GPIO 4`
- Права кнопка: `GPIO 18`
- Старт гри: натиснути й відпустити будь-яку кнопку на splash.
- Скидання рекорду: утримувати обидві кнопки 2 секунди на splash.
- Вихід із Game Over назад на splash: натиснути будь-яку кнопку.

## Апаратна частина

### Необхідні компоненти
- 1 x ESP32 DOIT DEVKIT V1
- 1 x OLED SSD1306 `128x64` I2C
- 2 x кнопки без фіксації
- Breadboard або макетна плата
- Джамперні дроти
- USB-кабель

### Схема підключення

![ESP32 Snake Wiring](assets/esp32-snake-wiring.svg)

Векторна схема: `assets/esp32-snake-wiring.svg`

#### OLED

| OLED | ESP32 |
|---|---|
| VCC | 3.3V |
| GND | GND |
| SDA | GPIO 21 |
| SCL | GPIO 22 |

#### Кнопки
- Сигнал лівої кнопки -> `GPIO 4`
- Сигнал правої кнопки -> `GPIO 18`
- Другий вивід кожної кнопки -> `GND`
- Логіка кнопок active-low

### Нотатки по монтажу
- Не подавайте `5V` напряму на GPIO ESP32.
- Живіть OLED від `3.3V`, якщо модуль не вимагає іншого.
- Тримайте дроти кнопок короткими, щоб зменшити хибні спрацювання.
- ESP32, OLED і кнопки мають мати спільну землю.

## Налаштування

### Збірка і прошивка
1. Встановіть VS Code.
2. Встановіть розширення PlatformIO IDE.
3. Склонуйте репозиторій:

```bash
git clone https://github.com/Matvik/esp_32_snake.git
```

4. Відкрийте проєкт у VS Code.
5. Дочекайтесь встановлення залежностей з `platformio.ini`.
6. Підключіть плату ESP32.
7. Зберіть прошивку:

```bash
pio run
```

8. Завантажте прошивку:

```bash
pio run --target upload
```

9. Відкрийте serial monitor за потреби:

```bash
pio device monitor -b 115200
```

### Native unit tests

У проєкті є окреме середовище `native` для desktop-тестів ядра логіки.

Запуск:

```bash
pio test -e native
```

Якщо команда `pio` не доступна у вашому shell, запускайте її з термінала VS Code, де активний PlatformIO.

## Архітектура

### Загальний потік виконання
- `src/main.cpp`: Arduino `setup()` / `loop()`.
- `src/Game.cpp`: стани гри, кнопки, рекорд, інтеграція core + renderer.
- `lib/snake_core`: чиста ігрова логіка.
- `lib/snake_render`: малювання на OLED і анімації.

### Модуль ядра

Файли:
- `lib/snake_core/include/SnakeCore.h`
- `lib/snake_core/src/SnakeCore.cpp`

Відповідає за:
- рух змійки
- wrap-around по межах
- перевірку колізій із тілом і перешкодами
- генерацію перешкод під час reset
- спавн їжі
- зміну швидкості гри
- детерміновану поведінку для тестів

### Модуль рендера

Файли:
- `lib/snake_render/include/SnakeRenderer.h`
- `lib/snake_render/src/SnakeRenderer.cpp`

Відповідає за:
- основний ігровий кадр
- splash-анімацію
- миготіння при зіткненні
- анімацію Game Over
- блимання їжі
- HUD зі score та high score

### Оболонка застосунку

Файли:
- `include/Game.h`
- `src/Game.cpp`

Відповідає за:
- переходи між станами `SPLASH`, `PLAYING`, `COLLISION`, `GAME_OVER`
- опитування кнопок через `ezButton`
- обробку довгого натискання
- збереження рекорду через `Preferences`
- зв'язування core та renderer

## Тести

Файл тестів:
- `test/test_snake_core/test_main.cpp`

Поточне покриття тестами:
- початковий стан після reset
- зміна напрямку
- рух і wrap-around
- колізія з тілом і перешкодою
- поїдання їжі та прискорення
- нижня межа швидкості
- безпечний спавн їжі
- валідність layout після reset
- інваріанти на довгих серіях кроків
- інваріанти при повторних reset

Тести сфокусовані на `SnakeCore`, бо саме там знаходяться правила гри і немає залежності від OLED чи кнопок.
